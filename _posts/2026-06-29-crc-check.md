## 什么是 CRC？

CRC（循环冗余校验）是数据链路层最常用的差错检测技术。发送方在数据末尾附加一个校验码，接收方用同样的多项式做除法，余数为 0 则数据无误。

## 计算步骤

1. 原始数据末尾补 n 个 0（n = 生成多项式位数 - 1）
2. 用生成多项式做模 2 除法（XOR 运算，不进位）
3. 余数即为 CRC 校验码
4. 发送帧 = 原始数据 + 校验码

## 例题

**数据：** 1101011011
**生成多项式：** 10011（x⁴ + x + 1，阶数 4）

**计算：**
1. 补 4 个 0：1101011011**0000**
2. 模 2 除法 → 余数：**1110**
3. 发送帧：**11010110111110**

**接收方验证：** 用 10011 除 11010110111110，余数为 0 ✅

## 常见生成多项式

| 名称 | 多项式 | 应用 |
|------|--------|------|
| CRC-8 | x⁸ + x² + x + 1 | 嵌入式 |
| CRC-16 | x¹⁶ + x¹⁵ + x² + 1 | USB |
| CRC-32 | x³² + x²⁶ + x²³ + ... | 以太网、ZIP |

> CRC 能检测所有奇数个错误、所有突发长度 ≤ 校验码位数的错误，但不能纠错——发现错误就要求重传。
