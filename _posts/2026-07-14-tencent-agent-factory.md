2026 年的腾讯，正在经历一场从产品公司到能力调度平台的转型。一批新的 Agent 产品密集上线，同时存量产品也在快速 Agent 化。

## 新 Agent 产品线

腾讯光子工作室推出了「造化工坊」——零门槛的 AI 互动游戏创作平台。此外还有 AI 设计工具 Ardot、创意设计产品 Miora、AI 漫剧创作平台 WorkRally、互动影视素材平台 TDream、数据工程平台 DataBuddy、自主学习平台 LearnBuddy。

这些产品覆盖了游戏、设计、影视、数据、教育等多个垂直领域，但共享同一个底层逻辑：将 AI 能力封装为可调用的 Agent 服务。

## WorkBuddy：当前最热门的出口

在这批新产品中，WorkBuddy 是增长最突出的一个。据易观分析报告，WorkBuddy 月访问量突破 885 万，是行业第二名的 2 到 3 倍，在国内桌面 Agent 赛道中日活最高。

WorkBuddy 的核心策略是将腾讯系所有能力以 Skill 形式接入。在「专家·技能·连接器」板块中，几乎整个腾讯的产品能力都被重新封装为可被 Agent 调度的模块。

## 存量产品的 Agent 化改造

更值得关注的是腾讯对存量产品的改造节奏。微信推出了 AI 助手小微，企业微信推出了 AI 助手大圆。腾讯文档、腾讯网盘、腾讯乐享三大办公产品被升级为更偏 Agent 原生的能力底座：

- 腾讯文档强调 AI 实时双写与人机协同
- 腾讯网盘强化 PB 级 AI 资产沉淀
- 腾讯乐享被放在企业知识基因库的位置上

## 真正的挑战：让 Agent 穿过腾讯

马化腾在今年股东大会上说了一段话：「原来一年前我们以为上了船，后来发现那个船漏水了，现在感觉站上去了，还坐不下去，还是希望船速能快一点。」

这句话反映了腾讯在 AI 浪潮中的真实处境。与字节跳动的激进打法不同，腾讯在过去两年里并不是最激进的玩家。但 2026 年的产品密集发布表明，战略方向已经明确。

腾讯真正的挑战不是做出一个 Agent，而是让 Agent 能够穿过腾讯——跨过部门边界、产品边界和数据边界，调用不同体系的能力，让社交、协作、内容、知识、云和工具这些原本分散的能力，以任务为中心被重新组织。

这背后需要解决的远不止模型效果，还包括接口统一、权限治理、数据协同、组织配合、商业分配和用户体验的一致性。

## 元宝的定位

腾讯确实需要元宝这样一个面向普通用户的独立 AI 产品，去承接搜索、问答、写作、总结等通用需求。但从长期优势看，元宝未必是最重要的那张牌。

腾讯原本就拥有中国互联网最稠密的一批入口——微信、企业微信、QQ、腾讯文档、腾讯会议、腾讯云、游戏、QQ 浏览器。真正有想象力的方向，是让 Agent 直接嵌入这些既有场景，而不是让用户单独打开一个新的 AI 应用。
