本文记录了我用 Hermes Agent 搭建个人知识管理系统的实际配置。这个系统每天自动抓取技术/AI/安全方向的最新文章，归档到本地知识库，筛选后发布到博客——整个过程不需要手动干预。

## 架构概览

系统由三个组件组成，通过 Hermes 的 cron 和 skill 机制串联：

```
RSS 抓取 (每天08:00) → 知识库归档 → 手动筛选 → 润色发布
    ↑                            ↓
  定时任务                     Hermes Agent
```

核心思路：让 Agent 承担信息收集和初筛的工作，人只做最后的判断和创作。

## 组件一：RSS 聚合抓取

每天早晨 8 点，一个 Python 脚本自动运行，抓取配置好的 RSS 源：

- **AI 方向**：少数派、腾讯云开发者
- **技术方向**：GitHub Blog、51CTO、小众软件
- **安全方向**：GitHub Security Blog

脚本按关键词过滤后，将匹配文章归档到 Obsidian 知识库的对应目录：

```
KB/学习资料/
├── AI学习/文章/     ← AI/LLM/Agent 相关
├── 技术文章/文章/   ← 开发/开源/工具
└── 网络安全/文章/   ← 安全/漏洞/渗透
```

同时生成搜索索引，供后续检索使用。

## 组件二：公众号文章归档

除了 RSS，还有另一个入口：微信公众平台的文章。通过 archive_wechat.py 脚本，发送一个链接就能自动完成：

1. 抓取文章内容
2. 提取标题、正文、日期
3. 下载图片到本地 assets
4. 按内容主题分类归档到 KB

这个脚本通过 Hermes skill 注册，用户发链接时自动触发。

## 组件三：从归档到发布

归档到 KB 的文章经过筛选后，由 Hermes Agent 完成润色和发布：

1. 读取原文，理解核心内容
2. 重新组织结构和表达方式，去掉原文痕迹
3. 添加原创分析和观点
4. 写入 Jekyll 博客的 _posts 目录
5. 自动 commit 和 push

整个过程不需要离开聊天界面。

## 定时任务配置

当前运行的定时任务：

| 任务 | 时间 | 说明 |
|------|------|------|
| RSS 聚合抓取 | 每天 08:00 | 自动抓取并归档 |
| 博客自动更新 | 每天 09:00/21:00 | 提交 uptime 记录 |
| 湖北半马监控 | 每天 01:00 | 检查报名状态 |

## 一些经验

**元数据与推理分离**：信息收集（RSS、网页抓取）用脚本完成，稳定且廉价。AI 只负责需要判断力的环节——筛选、润色、创作。这样能把 Token 花在最有价值的地方。

**知识库作为中间层**：所有抓取的内容先进 KB，而不是直接发博客。这样有一个缓冲期，可以沉淀、筛选，避免什么内容都往上堆。

**技能化重复操作**：公众号归档、文章润色这些重复流程都注册为 Hermes skill。下次执行时不需要重新描述需求，直接触发即可。
